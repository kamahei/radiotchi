## How to change a locked decision

1. Note the trigger (new RF data, a constraint discovered during implementation, a user
   request).
2. Add a new row or amend the entry with the new decision and rationale, and update any docs
   it affects in the same change.
3. If the change invalidates an acceptance criterion or a data-model invariant, update those
   too — do not let the docs drift.
